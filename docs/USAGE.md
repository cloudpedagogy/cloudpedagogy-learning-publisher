# Command Line Usage: CloudPedagogy Learning Publisher

The Learning Publisher provides a primary CLI tool called `coursegen` to manage the core authoring and building workflow. It also provides standalone Python scripts for generating specialized outputs like SCORM packages and PDF handbooks.

---

## The `coursegen` CLI

The `coursegen` command is the main entry point for the application.

### `init`
Creates a default course configuration template (`course.yml`) to help you get started quickly.
```bash
coursegen init --path config/my_course.yml
```

### `build`
Reads your YAML configuration file and generates the full Quarto directory scaffold (the `.qmd` files, `_quarto.yml`, navigation, etc.). This must be run *before* attempting to import any Word documents.
```bash
coursegen build config/my_course.yml
```
**Options:**
- `--force`: Overwrite existing `.qmd` files.
- `--output-dir`: Specify a custom output directory (defaults to `course/<module_id>`).

### `import_word`
The core of the Word-first workflow. It scans the YAML configuration for `.docx` paths, converts them using Pandoc, parses the custom syntax (quizzes, tabs, R code), and injects the formatted content into the Quarto scaffold generated by `build`.
```bash
coursegen import_word config/my_course.yml
```

### `render`
Invokes the Quarto rendering engine to compile the populated `.qmd` files into a final HTML website.
```bash
coursegen render config/my_course.yml
```

### `preview`
Renders a single `.qmd` page to a temporary `.preview` directory for rapid authoring feedback without needing to render the entire site.
```bash
coursegen preview course/my_course/se01/01-intro/intro1.qmd
```

### `validate`
Performs a structural validation of your YAML configuration file to ensure it matches the required schema before attempting to build.
```bash
coursegen validate config/my_course.yml
```

### `inspect`
Prints a human-readable summary of the course structure defined in your YAML configuration to the console.
```bash
coursegen inspect config/my_course.yml
```

## Standalone Tools

The repository contains standalone Python scripts in the `src/course_generator/tools/` directory for generating specialized educational outputs.

### SCORM Packager
Wraps your fully rendered Quarto HTML website inside a SCORM 1.2 manifest, injecting a JavaScript API wrapper to track completion within a Learning Management System (LMS) like Moodle or Canvas.
```bash
python src/course_generator/tools/package_scorm.py \
    --site-dir output/my_course \
    --title "My Course Title" \
    --output-dir dist_scorm
```
This generates a ZIP file ready for LMS upload in the `dist_scorm` directory.

### Handbook Generator
Parses the `_quarto.yml` configuration to determine the reading order of the course. It then concatenates all `.qmd` files, strips out interactive web elements that do not work in print (like tabs, HTML forms, and embeds), and generates a single continuous handbook.
```bash
python src/course_generator/tools/build_handbook_from_quarto.py course/my_course
```
This command will output both a PDF (if TinyTeX is installed) and a Word Document representation of your entire course.
